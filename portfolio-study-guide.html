1) Project Goal

Build an interactive personal portfolio in React that contains your photo, about section, hobbies, education, projects, skills, and contact details.

3) Core React Concepts You Used

  • useState: stores values that can change in UI (dark mode, selected skill category).
  • useMemo: calculates filtered skills efficiently when category changes.
  • map(): loops through arrays to render hobbies, education, projects, and skills.
  • Conditional class: adds dark class when dark mode is true.

4) Important Code - State and Data

import { useMemo, useState } from 'react'

const [activeSkillCategory, setActiveSkillCategory] = useState('All')
const [darkMode, setDarkMode] = useState(false)

const profile = {

Why this matters: React re-renders UI when state changes. Clicking buttons changes state, and UI updates automatically.

5) Important Code - Interactive Skill Filter

const skillGroups = {
  Languages: ['Java', 'JavaScript', 'TypeScript'],
  Web: ['React.js', 'Node.js', 'Express.js', 'HTML', 'CSS'],
}

const filteredSkills = useMemo(() => {
  if (activeSkillCategory === 'All') {
    return Object.entries(skillGroups).flatMap(([group, skills]) =>
      skills.map((skill) => ({ group, skill })),
    )
  }
  return skillGroups[activeSkillCategory].map((skill) => ({
    group: activeSkillCategory,
    skill,
  }))
}, [activeSkillCategory])

How it works: button click changes activeSkillCategory, then filteredSkills recomputes and the visible list changes.

6) Important Code - Rendering Lists with map()

{projects.map((project) => (
  <article key={project.title} className="project-card">
    <h4>{project.title}</h4>
    <p>{project.details}</p>
    <span>{project.stack}</span>
  </article>
))}

Rule: Always provide a unique key in mapped elements to help React track items.

7) Important Code - Dark Mode Toggle

<div className={`portfolio ${darkMode ? 'dark' : ''}`}>
  <button onClick={() => setDarkMode((prev) => !prev)}>
    {darkMode ? 'Light' : 'Dark'} Mode
  </button>
</div>

Idea: one boolean state controls theme class. CSS handles how dark styles look.

8) CSS Architecture Used

  • Global styles in src/index.css (font, margin reset, smooth scroll).
  • Component styles in src/App.css (cards, grid, pills, nav, dark theme).
  • Responsive design using media query for mobile screens.

9) How to Run and Study Step-by-Step

  1. Run project with npm run dev.
  2. Open src/App.jsx.
  3. Read from top to bottom: imports - states - data arrays - JSX return.
  4. Change one thing (example: add one skill) and see live update in browser.
  5. Toggle dark mode and inspect class change in browser devtools.
